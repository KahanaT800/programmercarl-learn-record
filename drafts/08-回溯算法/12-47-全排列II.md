### 看完题解后的想法
重排序的当前层去重有点绕；

去重条件 1 :

```cpp
nums[i] == nums[i-1] && !used[i-1]
```
 
- `nums[i] == nums[i-1]`：当前值和前一个相同
- `!used[i-1]`：前一个值**没在用**（已被回撤），说明前一个是**同层**之前的循环中选过又撤销的 → **同层重复，跳过**
- `used[i-1] == true`：前一个值**正在用**（是上层选的），当前层再选是合法的 → **不跳过**
 
完整递归树展开：`nums = [1, 1, 2]`
 
```text
第一层 (index 无关，从头遍历)
│
├─ i=0, 选 nums[0]=1, used=[T,F,F]
│  │
│  ├─ i=0: used[0]=T → 跳过
│  ├─ i=1: nums[1]==nums[0], 但 used[0]=T（正在用）→ 不跳过
│  │  选 nums[1]=1, used=[T,T,F]
│  │  │
│  │  ├─ i=0: used[0]=T → 跳过
│  │  ├─ i=1: used[1]=T → 跳过
│  │  └─ i=2: 选 nums[2]=2 → path=[1,1,2] ✅ 收集
│  │
│  └─ i=2: 选 nums[2]=2, used=[T,F,T]
│     │
│     ├─ i=0: used[0]=T → 跳过
│     ├─ i=1: 选 nums[1]=1 → path=[1,2,1] ✅ 收集
│     └─ i=2: used[2]=T → 跳过
│
├─ i=1, nums[1]==nums[0], used[0]=F（没在用，已回撤）→ 同层重复 → 跳过 ❌
│  （整棵子树被剪掉！）
│
└─ i=2, 选 nums[2]=2, used=[F,F,T]
   │
   ├─ i=0: 选 nums[0]=1, used=[T,F,T]
   │  │
   │  ├─ i=1: nums[1]==nums[0], used[0]=T（正在用）→ 不跳过
   │  │  选 nums[1]=1 → path=[2,1,1] ✅ 收集
   │  └─ i=2: used[2]=T → 跳过
   │
   └─ i=1: nums[1]==nums[0], used[0]=F → 同层重复 → 跳过 ❌
 
结果：[1,1,2], [1,2,1], [2,1,1] ✅
```
 
**关键**：第一层 `i=1` 直接被剪掉，整棵子树不展开，效率高。

去重条件2 同枝去重

```cpp
nums[i] == nums[i-1] && used[i-1]
```
 
- `used[i-1] == true`：前一个值**正在用**（是上层选的），此时跳过 → 禁止相同值**按下标顺序连续使用**
- `!used[i-1]`：前一个值没在用 → 不跳过
 
完整递归树展开：`nums = [1, 1, 2]`
 
```
第一层
│
├─ i=0, 选 nums[0]=1, used=[T,F,F]
│  │
│  ├─ i=0: used[0]=T → 跳过
│  ├─ i=1: nums[1]==nums[0], used[0]=T → 同枝重复 → 跳过 ❌
│  └─ i=2: 选 nums[2]=2, used=[T,F,T]
│     │
│     ├─ i=0: used[0]=T → 跳过
│     ├─ i=1: 选 nums[1]=1 → path=[1,2,1] ✅ 收集
│     └─ i=2: used[2]=T → 跳过
│
├─ i=1, nums[1]==nums[0], used[0]=F → 不跳过（没在用）
│  选 nums[1]=1, used=[F,T,F]    ← 注意：这棵子树完整展开了！
│  │
│  ├─ i=0: nums[0]==nums[1], used[1]=T → 跳过 ❌
│  │  （这里阻止了 [1,1,...] 的产生... 但在更深处才拦住）
│  └─ i=2: 选 nums[2]=2, used=[F,T,T]
│     │
│     └─ i=0: 选 nums[0]=1 → path=[1,2,1]... 
│        但等等，这里 nums[0]==nums[1], used[1]=T → 跳过？
│        不，i=0 时检查的是 i>0，i=0 不满足 → 不跳过
│        → path=[1,2,1] → 和上面重复了？
│
│  实际上需要更仔细地检查...
│  i=0: i>0 不成立，不触发去重 → 选 nums[0]=1, used=[T,T,F]
│  │  i=2: 选2 → path=[1,1,2] ✅ 收集
│  i=2: 选 nums[2]=2, used=[F,T,T]
│     i=0: i>0 不成立 → 选 nums[0]=1 → path=[1,2,1] ✅
│     → 但 [1,2,1] 已经在 i=0 的子树里产生过了？
│
│  最终去重条件在更深层保证了不会真正重复收集，
│  但大量分支被展开后才被拦住。
│
└─ i=2, 选 nums[2]=2, used=[F,F,T]
   │
   ├─ i=0: 选 nums[0]=1, used=[T,F,T]
   │  ├─ i=1: nums[1]==nums[0], used[0]=T → 跳过 ❌
   │  └─ 无可选 → 死路
   │
   └─ i=1: nums[1]==nums[0], used[0]=F → 不跳过
      选 nums[1]=1, used=[F,T,T]
      │
      └─ i=0: 选 nums[0]=1 → path=[2,1,1] ✅ 收集
 
结果：[1,2,1], [1,1,2], [1,2,1]?... 最终结果仍然正确，
但中间展开了更多分支。
```

### 代码

```cpp
class Solution {
public:
    vector<vector<int>> res;
    vector<int> path;
    void backtracking(vector<int>& nums, vector<bool>& globalSet) {
        if (path.size() == nums.size()) {
            res.push_back(path);
            return;
        }

        for (int i = 0; i < nums.size(); i++) {
            if (globalSet[i]) continue; // 全局去重
            if (i > 0 && nums[i] == nums[i-1] && !globalSet[i-1]) continue; // 当前层去重
            globalSet[i] = true;
            path.push_back(nums[i]);
            backtracking(nums, globalSet);
            globalSet[i] = false;
            path.pop_back();
        }
    }
    vector<vector<int>> permuteUnique(vector<int>& nums) {
        sort(nums.begin(), nums.end());
        vector<bool> globalSet(nums.size(), false);
        backtracking(nums, globalSet);
        return res;
    }
};
```
